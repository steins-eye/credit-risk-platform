# 信用风险管理系统用例描述

## 4 用例描述

### 4.1 买家用例

#### 4.1.1 商品浏览与下单
**前置条件：**
- 买家已登录系统
- 商品已上架且状态正常

**后置条件：**
- 生成待支付订单
- 订单状态为"待支付"

**业务前端子系统流程：**
1. 买家浏览商品列表，查看商品详情
2. 选择商品规格，查看信用优惠信息
3. 点击下单，确认订单信息
4. 进入支付页面，显示支付时限

**业务后端子系统流程：**
1. 验证商品状态和库存
2. 计算订单金额（含信用优惠）
3. 生成订单记录，状态为"待支付"
4. 启动支付时限倒计时

**评价决策子系统流程：**
1. 根据买家信用分计算可享受的信用优惠
2. 评估交易风险等级

#### 4.1.2 订单支付
**前置条件：**
- 存在待支付订单
- 订单未超时

**后置条件：**
- 订单状态更新为"已支付"
- 生成卖家账户流水
- 若含押金，生成押金流水

**业务前端子系统流程：**
1. 买家选择支付方式
2. 完成支付操作
3. 显示支付成功页面

**业务后端子系统流程：**
1. 处理支付请求，验证支付结果
2. 更新订单状态为"已支付"
3. 生成卖家账户流水，状态为"已结算"
4. 若商品含押金，生成押金流水，状态为"冻结中"

#### 4.1.3 即时通讯咨询
**前置条件：**
- 买家已登录
- 商品页面可访问

**后置条件：**
- 生成聊天记录
- 消息已发送给卖家

**业务前端子系统流程：**
1. 买家在商品页面点击咨询
2. 打开聊天窗口
3. 输入消息并发送

**业务后端子系统流程：**
1. 建立实时通信连接
2. 存储聊天记录
3. 推送消息给卖家

#### 4.1.4 申请售后退款
**前置条件：**
- 订单已完成支付
- 订单处于可售后状态

**后置条件：**
- 生成售后单，状态为"待卖家处理"
- 关联订单信息

**业务前端子系统流程：**
1. 买家在订单详情页面点击申请售后
2. 选择退款类型（订单金额/押金金额）
3. 填写退款原因
4. 提交售后申请

**业务后端子系统流程：**
1. 验证订单售后资格
2. 生成售后单，状态为"待卖家处理"
3. 通知卖家处理售后申请

#### 4.1.5 提交纠纷单
**前置条件：**
- 售后单状态为"已拒绝"
- 买家对拒绝结果不满意

**后置条件：**
- 生成纠纷单
- 纠纷单进入仲裁流程

**业务前端子系统流程：**
1. 买家在售后单页面点击提交纠纷
2. 填写纠纷详情，上传证据
3. 提交纠纷申请

**业务后端子系统流程：**
1. 验证售后单状态
2. 生成纠纷单，包含完整订单信息和聊天记录
3. 调用评价决策系统进行初步裁决

**评价决策子系统流程：**
1. 分析订单信息、聊天记录等数据
2. 评估是否需要人工介入
3. 给出初步裁决结果

### 4.2 卖家用例

#### 4.2.1 卖家入驻
**前置条件：**
- 用户未注册为卖家
- 系统允许新卖家入驻

**后置条件：**
- 卖家账户创建成功
- 初始信用分设置完成
- 押金计算并显示

**业务前端子系统流程：**
1. 用户选择注册为卖家
2. 填写卖家基本信息
3. 选择经营类目（可多选）
4. 查看系统计算的押金金额
5. 确认入驻

**业务后端子系统流程：**
1. 验证信息完整性
2. 设置初始信用分
3. 根据经营类目计算应缴押金（取最大类目押金）
4. 创建卖家账户
5. 初始化卖家权限和功能

#### 4.2.2 商品上架
**前置条件：**
- 卖家已成功入驻
- 卖家账户状态正常

**后置条件：**
- 商品信息提交成功
- 商品进入审核流程，状态为"待审核"

**业务前端子系统流程：**
1. 卖家进入商品管理页面
2. 点击上架新商品
3. 填写商品基本信息（描述、图片、规格）
4. 设置信用优惠信息（租赁类可设信用免押额度）
5. 提交商品审核

**业务后端子系统流程：**
1. 接收商品信息
2. 生成商品记录，状态为"待审核"
3. 调用审核流程

#### 4.2.3 处理售后申请
**前置条件：**
- 存在待处理的售后单
- 卖家已登录

**后置条件：**
- 售后单状态更新
- 若同意退款，触发自动退款流程

**业务前端子系统流程：**
1. 卖家查看售后申请列表
2. 审核售后申请详情
3. 选择同意或拒绝
4. 填写处理意见

**业务后端子系统流程：**
1. 更新售后单状态
2. 若卖家同意：
   - 状态改为"待买家处理"
   - 如需退款，触发自动退款流程
3. 若卖家拒绝：
   - 状态改为"已拒绝"
   - 通知买家处理结果

#### 4.2.4 押金解冻
**前置条件：**
- 订单已完成
- 存在冻结中的押金流水
- 卖家确认收到货物

**后置条件：**
- 押金流水状态改为"已退还"
- 买家押金解冻

**业务前端子系统流程：**
1. 卖家在订单管理页面查看已完成订单
2. 确认收到货物
3. 点击解冻押金

**业务后端子系统流程：**
1. 验证订单状态
2. 更新押金流水状态为"已退还"
3. 解冻买家押金

### 4.3 仲裁员用例

#### 4.3.1 商品审核
**前置条件：**
- 存在待审核的商品
- 仲裁员已登录系统

**后置条件：**
- 商品审核状态更新
- 卖家信用分相应调整

**业务前端子系统流程：**
1. 仲裁员查看待审核商品列表
2. 查看商品详细信息
3. 查看AI审核结果和建议
4. 做出审核决定（通过/不通过）
5. 填写审核意见

**业务后端子系统流程：**
1. 提供待审核商品数据
2. 记录审核结果
3. 更新商品状态：
   - 审核通过：商品标记为"已上架"
   - 审核不通过：商品标记为"审核失败"
4. 根据审核结果调整卖家信用分

**评价决策子系统流程：**
1. **第一层：关键词与正则过滤**
   - 使用敏感词库（政治、违禁品、暴恐）进行秒级拦截
   - 直接在后端接口层处理

2. **第二层：AI内容识别**
   - 图片识别：检测二维码引流、涉黄、违禁品Logo
   - 文本识别：检测虚假宣传（如"最"、"第一"）
   - AI判断审核是否通过、是否需要人工介入

3. **第三层：人工复审**
   - 医疗器械、食品等预设类目强制要求人工审核
   - 提供AI审核结果作为参考

#### 4.3.2 纠纷处理
**前置条件：**
- 存在需要人工处理的纠纷单
- AI已给出初步裁决建议

**后置条件：**
- 纠纷单状态更新
- 裁决结果生效
- 信用分相应调整

**业务前端子系统流程：**
1. 仲裁员查看待处理纠纷单列表
2. 查看纠纷详情（订单信息、聊天记录等）
3. 查看AI裁决信息和建议
4. 做出最终裁决
5. 填写裁决理由

**业务后端子系统流程：**
1. 提供纠纷单完整信息
2. 记录仲裁员裁决结果
3. 更新纠纷单状态
4. 根据裁决结果：
   - 卖家胜诉：流程结束
   - 买家胜诉：售后单状态改为"待买家处理"，触发退款流程
5. 调整相关方信用分

**评价决策子系统流程：**
1. 分析纠纷单相关数据
2. 评估纠纷复杂程度
3. 判断是否需要人工介入
4. 提供裁决建议和风险评估

### 4.4 管理员用例

#### 4.4.1 系统配置管理
**前置条件：**
- 管理员已登录系统
- 具有系统配置权限

**后置条件：**
- 系统配置参数更新
- 配置生效

**业务前端子系统流程：**
1. 管理员进入系统配置页面
2. 查看和修改各项配置参数
3. 保存配置更改

**业务后端子系统流程：**
1. 提供当前系统配置
2. 验证配置参数合法性
3. 更新系统配置数据库
4. 刷新系统缓存，使新配置生效

**配置项包括：**
- 卖家和买家的初始信用分
- 卖家/买家信用分标签设置
- 订单支付时限
- 商品类目信息
- 敏感词库管理
- AI审核参数配置

#### 4.4.2 用户账户管理
**前置条件：**
- 管理员已登录系统
- 具有用户管理权限

**后置条件：**
- 用户账户状态更新
- 相关权限调整

**业务前端子系统流程：**
1. 管理员查看用户账户列表
2. 搜索和筛选特定用户
3. 查看用户详情和操作记录
4. 执行账户管理操作（冻结、解冻、权限调整等）

**业务后端子系统流程：**
1. 提供用户账户数据
2. 执行账户状态变更
3. 记录管理操作日志
4. 通知相关用户状态变更
